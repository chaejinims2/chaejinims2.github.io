1️⃣ 리눅스 부팅 순서 2️⃣ runlevel 의미 3️⃣ init vs systemd 차이 4️⃣ systemctl 명령어

Kernel

      ↓
서비스 실행

1. runlevel 개념 (SysV init)

runlevel은 시스템의 운영 상태(level) 를 의미합니다.

범위

0 ~ 6

각 숫자는 시스템 동작 모드를 나타냅니다.

Runlevel 의미
0 시스템 종료
1 단일 사용자 모드
2 다중 사용자 (네트워크 없음)
3 다중 사용자 (CLI)
4 사용자 정의
5 GUI 모드
6 재부팅

시험에서 가장 중요한 것은 0,1,3,5,6 입니다.

2. systemd target 개념

systemd에서는 runlevel 대신 target을 사용합니다.

target은 서비스 그룹(상태) 을 의미합니다.

예

multi-user.target
graphical.target

3. runlevel ↔ target 매핑

시험에서 가장 자주 묻는 표입니다.

Runlevel systemd Target
0 poweroff.target
1 rescue.target
2 multi-user.target
3 multi-user.target
4 multi-user.target
5 graphical.target
6 reboot.target

특히 시험에서 많이 묻는 것:

3 → multi-user.target
5 → graphical.target

runlevel

6. target 변경

CLI 모드

systemctl set-default multi-user.target

GUI 모드

systemctl set-default graphical.target

7. 즉시 target 변경

예

systemctl isolate multi-user.target

즉시 runlevel 변경과 유사합니다.
